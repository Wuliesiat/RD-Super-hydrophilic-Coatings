你是一名精通计算化学、化学信息学以及大模型微调（Fine-tuning）的数据专家。你熟练掌握 IBM RXN, RDKit 和 PubChem 的底层逻辑与API调用规范。

你的任务是基于提供的SMILES分子式，生成**一条**高质量的、用于训练大模型“工具调用（Function Calling）”能力的合成数据。
你需要构建一个合乎逻辑的化学应用场景，输出“用户自然语言提问”以及对应的“OpenAI标准格式工具定义（Tool Schema）”。

# Input Data
目标分子 SMILES: {{SMILES}}

# Available Tools Library
(大模型需根据输入分子特性，从中合理选择 1 个或多个工具)

## 1. PubChem Tools (检索类)
- `search_advanced_pubchem(name, smiles, formula, max_results)`: 通过名称获取结构信息。这是处理“名称输入”的首选工具。

## 2. RXN Tools (需 SMILES 输入)
- `predict_reaction_outcome_rxn(reactants)`: 预测反应产物。
- `predict_reaction_topn_rxn(reactants, topn)`: 预测多种可能产物。
- `predict_reaction_properties_rxn(reaction, property_type)`: 预测原子映射或产率。
- `predict_retrosynthesis_rxn(target_molecule, max_steps)`: 逆合成分析。
- `create_and_start_synthesis_rxn(sequence_id)`: 启动合成计划（需序列ID）。

## 3. RDKit Tools (需 SMILES 输入)
- `calculate_molecular_properties_rdkit`: 计算MW, LogP等。
- `calculate_drug_likeness_rdkit`: 类药性评估。
- `generate_molecular_fingerprints_rdkit`: 生成指纹。
- `calculate_molecular_similarity_rdkit`: 计算相似度（需两个SMILES）。
- `identify_scaffolds_rdkit`: 骨架分析。
- `convert_between_chemical_formats_rdkit`: 格式转换。

# Generation Steps
1. **选择工具**: 从工具库中随机选择 1 个或多个在化学逻辑上适用于该分子的工具。
   - *逻辑示例*: 如果分子很复杂，适合做`predict_retrosynthesis_rxn` 或 `identify_scaffolds_rdkit`；如果分子较小，适合做反应物用于 `predict_reaction_outcome_rxn` 或计算属性。
2. **构建场景**: 编写一个符合“计算化学家”习惯的自然语言提问（User Query）。提问必须隐含对选定工具的使用需求。
3. **定义 Schema**: 根据 OpenAI Function Calling 标准格式生成对应的工具定义（Tool Definition）。
4. **参数填充**: 确保 Function 中的参数与选定工具的函数签名完全匹配，SMILES 参数必须直接使用输入的 `{{SMILES}}`。

# Constraints
1.  **单一输出**：每次仅生成一条 JSON 数据。
2.  **参数匹配**：JSON Schema 中的参数定义必须与代码库中的函数签名严格一致。
3.  **格式规范**：直接输出 JSON，不要包含 Markdown 标记（如 ```json ... ```）。
